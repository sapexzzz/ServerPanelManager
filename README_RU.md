# Modrinth Extractor

Скрипт для скачивания модов, ресурспаков, шейдеров и датапаков из файла `modrinth.index.json` (экспорт сборки Modrinth / Prism Launcher).

## Требования

- Python 3.8+
- Пакет `requests`

```bash
pip install requests
```

## Использование

```bash
python extract.py modrinth.index.json
```

### Аргументы

| Аргумент | По умолчанию | Описание |
|---|---|---|
| `json_file` | — | Путь к `modrinth.index.json` |
| `-o`, `--output` | `extract` | Папка для скачанных файлов |
| `--retries` | `3` | Кол-во повторных попыток при ошибке сети |
| `--skip-hash` | выкл. | Пропустить проверку SHA1/SHA512 |

### Примеры

```bash
# Скачать в папку extract/ рядом со скриптом
python extract.py modrinth.index.json

# Указать другую папку назначения
python extract.py modrinth.index.json -o ~/Downloads/my_modpack

# 5 попыток, без верификации хеша
python extract.py modrinth.index.json --retries 5 --skip-hash
```

## Структура выходной папки

```
extract/
├── mods/           # .jar — моды
├── resourcepacks/  # .zip — ресурспаки
├── datapacks/      # датапаки
├── shaders/        # шейдеры
└── other/          # всё остальное
```

Категория определяется по **пути** в JSON (приоритет), затем по расширению и имени файла.

## Что исправлено по сравнению с оригинальным .sh

| Проблема | Решение |
|---|---|
| `pkg install` — Termux-специфично | Только стандартный `requests` |
| `while read` в subshell → счётчик всегда 0 | Python-цикл без subshell |
| Нет повторных попыток при сбое | Экспоненциальный backoff (3 попытки) |
| Хеш проверялся даже при сбое wget | Хеш проверяется только после успешной загрузки |
| Только SHA1 | SHA512 → SHA1 (приоритет по стандарту Modrinth) |
| Нет кеширования | Уже скачанные файлы пропускаются |
| Неверная категория для датапаков | Детекция по пути + расширению + имени |
| Неполный файл остаётся на диске | Удаляется при сбое или несовпадении хеша |
| Нет итоговой сводки ошибок | Выводится список файлов с ошибками |
